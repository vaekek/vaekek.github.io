Codesplitting in react

In React können wir Components welche erst zu gewissem Zeitpunkt benötigt werden outsourcen und mit react-loadable erst dann laden, wenn diese wirklich benötigt werden.
Um dies nutzen zu können wird die Library ​react-loadable​ benötigt. 
Diese kann mit folgenden Befehl installiert werden: ​npm i react-loadable
Hier ein Beispiel:
import Loadable from 'react-loadable';
import Loading from './my-loading-component';
const LoadableComponent = Loadable({
loader: () => import('./my-component'), // welche Component erst importiert werden soll wenn diese genutzt wird
loading: Loading, // Welche Component angezeigt werden soll, bis die gewünschte Component gerendert wird
});
export default class App extends React.Component {
render() {
return <LoadableComponent/>;
}
}
Dies ist nützlich wenn Components mit Links auf andere Components verweisen, z.B. in einer NavBar, dann wird nur die initial Component geladen und per Klick auf die Navbar, die gewünschte Component für den jeweiligen Link nachgeladen sobald diese benötigt wird.


Wann hat es einen Nutzen?

Es ist eine Balance ab wann es wirklich nötig ist, Components nachzuladen. 
Kleine Components (< 20kB zipped) müssen nicht nachgeladen werden. Das ganze sollte auch mit der Netzwerklatenz abgewogen werden.

Wir wird es noch schneller?

ServiceWorkers! Initial wird nur die benötigte Page geladen und der ServiceWorker ladet im Hintergrund alle nötigen Files und cached diese, damit wird die Ladezeit reduziert, sobald das File benötigt wird.
