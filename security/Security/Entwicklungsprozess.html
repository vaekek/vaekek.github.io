Entwicklungsprozess

Wer muss sicher entwickeln?

  • Grundkenntnisse in der sicheren Softwareentwicklung sollte jeder Entwickler haben
  • Experten für Spezialthemen wird es aber immer benötigen

Die Grösse eines Unternehmens und die erwartete Benutzerzahl sollte nie ein Kriterium sein, ob Software sicher entwickelt wird oder nicht.

Entwicklungsprozess

Wer bezahlt den Preis einer sicheren Webanwendung?

Schutz vor Standardbedrohungen wie SQL Injektion, XSS etc. müssen eigentlich im Preis enthalten sein, da Mehraufwand nicht sehr hoch ist.

Mehr Aufwand benötigen sicher:
  • Spezifikationen(Validierungsregeln, Bedrohungsmodelle, …)
  • Anbindung eines Benutzerverzeichnisses
  • Rechte -/ und Rollenverwaltung
Hier kann kein pauschaler Mehraufwand einberechnet werden, der Aufwant muss geschätzt werden.

Sicherheit ist kein einmaliges Thema

Gegenmassnahmen gegen potenzielle Angreifer müssen über die Lebensdauer einer Software garantiert angepasst werden.

Damit Sicherheit nicht vernachlässigt wird, sollte sie in den Entwicklungsprozess einer Applikation integriert sein.
  • Bedeutet Veränderungen im Entwicklungsprozess, dass sich auch alle Phasen um das Thema Sicherheit kümmern.

Klärung der notwendigen Sicherheitsanforderungen

  • Standardabsicherung
  • Threat Modeling mit Trust Boundaries
  • Benutzer und Rollen
  • Authentifizierung / Autorisierung

Authentisierung =    Eingabe von Login-Daten (Behauptung einer Identität)    
Authentifizierung =  Überprüfung der Behauptung durch das System (Verifizierung der Authentisierung)
Autorisierung =        Prüfung der Rechte und Konsequenz (Einräumung oder Verweigerung von Rechten)

Risikoanalyse

Vorteile: 
  • Bewusstsein über betroffene Applikationsteile
  • Gegenmassnahmen bereits überlegt und können sofort eingeleitet werden

Sicherheit einplanen

Die Einbindung der Sicherheitsmassnahmen sollte in den Entwicklungsprozess eingebunden werden.
